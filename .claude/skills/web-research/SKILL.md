# Web Research

タスク実行中に最新の技術情報を取得するためのワークフロー。

## 調査手順

### Step 1: WebSearchで候補を検索

以下の情報源を優先的に検索する：

- 公式サイトのリリース情報（例: `Node.js releases site:nodejs.org`）
- endoflife.date でのサポート状況（例: `Python endoflife.date`）
- GitHub リリースページ（例: `React releases site:github.com`）
- Docker Hub（コンテナイメージの場合）

### Step 2: WebFetchで詳細を取得

WebSearchの結果から信頼できるURLを選び、以下を確認する：

- サポート期間・EOL日
- LTS / Stable の区分
- 破壊的変更の有無
- 推奨バージョン

## 出力フォーマット

調査結果は以下の形式で報告する：

```
| 項目 | 値 |
|------|-----|
| 技術名 | X |
| 最新安定版 | X.Y.Z |
| LTS版 | X.Y.Z（あれば） |
| サポート終了日 | YYYY-MM-DD |
| 情報源 | [ソース名](URL) |
```

根拠として、情報源からの引用（原文）を含める。

## 注意事項

- 必ず公式ソースまたは信頼できる情報源を参照する
- 根拠が不明確な場合はその旨を明記する
- 情報の取得日時を意識し、古い情報には注意喚起する
- 複数の情報源で矛盾がある場合は公式ドキュメントを優先する
